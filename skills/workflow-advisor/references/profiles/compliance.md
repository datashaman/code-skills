# Profile: compliance

The heaviest profile. Opt-in only. Required for regulated environments
(SOC2, HIPAA, GDPR, PCI). Forces multiple defaults to flip — notably,
the decisions log moves from gitignored to committed.

## Contributions

**Artifacts.**
- `compliance_assessment` — for pii, payments, regulated, audit_relevant.
  Approval from `legal_compliance`.
- `data_handling_record` — for pii, gdpr_relevant. Approval from
  `legal_compliance` and `security`.
- `audit_trail_evidence` — auto-generated. Lives in `docs/compliance/audit/`.

**Roles.**
- `legal_compliance` — approves assessments, attests releases.
- `auditor` — read-only audit access. Cannot dispatch state-changing
  actions; can only export and consume.

**Labels.** `compliance:assessed`, `compliance:attested`, `audit:retained`,
`needs:compliance-assessment`, `needs:attestation`.

**Slash commands.** `/approve-compliance`, `/attest [artifact-id]`,
`/audit-export [range]`.

**Frameworks.**
```yaml
compliance:
  frameworks: [soc2, hipaa, gdpr, pci_dss]
```

The frameworks field determines which evidence is required. Each
framework declares its required artifacts and retention rules.

## Defaults this profile flips

When `compliance.enabled: true` AND `frameworks` includes audit-requiring
ones (any of soc2, hipaa, pci_dss), bootstrap automatically:

1. Removes `decisions/` from `.workflow/.gitignore` (must be committed
   for audit retention).
2. Removes `metrics/events.jsonl` from `.workflow/.gitignore`.
3. Sets `archive.retention: forever`.
4. Sets attestation requirements on releases.
5. Adds `auditor` role with `access_mode: read_only`.

The skill warns the team prominently on activation: "Enabling compliance
with audit-requiring frameworks will commit your decisions log and metrics
events to the repo. This is required for audit but means historic
process discussion will be in git history. Acknowledge to proceed."

## Attestations

Compliance differs from other profiles in requiring named, timestamped,
immutable attestations rather than just approvals.

`/attest {artifact-id}` writes:
```yaml
# .workflow/artifacts/compliance/attestations/2026-05-09-rel-1.2.0-marlin.yml
artifact: rel-1.2.0
attestation_type: release_compliance
attester: { handle: marlin, role: legal_compliance }
attested_at: 2026-05-09T14:23:00Z
content_hash: sha256:...      # of the artifact at attestation time
attester_acknowledgement: |
  I attest that release 1.2.0 has been assessed against SOC2 controls
  CC6.1, CC6.2, CC6.3 and meets requirements.
```

Attestations are append-only. Editing one creates a new attestation
linked to the prior; the prior is never overwritten.

## Audit trail

Every reconcile pass that writes to `.workflow/` while compliance is
active also writes a structured entry to `audit_trail_evidence`:

```yaml
- ts: 2026-05-09T14:23:00Z
  event: artifact.update_state
  actor: marlin
  artifact: spec/0042
  before_state: in-review
  after_state: approved
  reconcile_commit: a3f2c0
  decision_log_ref: 2026-05-09.md#decision-12
```

The audit trail is generated by the skill, never hand-edited. It links
back to the git commit and decision log entry for full provenance.

## Read-only auditor role

The `auditor` role can:
- Run reports.
- Use `/audit-export` to generate auditor-ready exports.
- Read all sidecars and artifacts.

The auditor cannot:
- Approve artifacts.
- Override gates.
- Skip stages.

This is enforced by `role.check_authority` checks in every state-changing
slash command.

## Gates

| Stage transition | Gate |
|---|---|
| impl-plan → implementation | `compliance_assessment_drafted_if_required` |
| review → merge-ready | `compliance_assessment_approved`, `attestations_collected_if_required` |
| released → validated | `audit_trail_complete` |

## Retention

```yaml
compliance:
  retention:
    audit_artifacts: 7_years
    decision_logs:   7_years   # overrides the default gitignore behavior
```

When `archive.migration_threshold` triggers, compliance teams should
migrate to a permanent docs location rather than pruning. The skill
prompts accordingly when compliance is active.
